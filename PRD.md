## S-1. Executive Summary

**SoulMatch는 Browser Use 자율 웹 에이전트가 Instagram에서 시작해 LinkedIn, Facebook, Twitter/X까지 크로스 플랫폼으로 탐색하며, 한 사람의 웹 상 모든 흔적을 수집·분석하여 완벽한 "디지털 트윈"을 생성하는 소셜 인텔리전스 플랫폼이다.**

사람들의 온라인 정체성은 여러 플랫폼에 조각조각 흩어져 있다. Instagram에는 라이프스타일, LinkedIn에는 커리어, Facebook에는 인맥, Twitter에는 생각. 어떤 도구도 이걸 하나로 모아 "이 사람이 누구인지"를 완전히 이해하지 못한다.

SoulMatch는 이 문제를 **크로스 플랫폼 자율 웹 에이전트**로 해결한다:

| 단계      | 기존 방식                         | SoulMatch                                                           |
| --------- | --------------------------------- | ------------------------------------------------------------------- |
| 정보 수집 | 하나의 플랫폼, 수동 검색          | Browser Use가 IG → 이메일/링크 추출 → LinkedIn/FB/Twitter 자동 탐색 |
| 사람 이해 | 프로필 사진 몇 장, 자기소개 3줄   | 멀티 플랫폼 데이터 통합 분석 → **디지털 트윈** 생성                 |
| 매칭/접근 | 외모 기반 스와이프, 매스 콜드메일 | 디지털 트윈 vs 디지털 트윈 유사도 매칭 + 개인화 아웃리치            |
| 실행      | 직접 DM 작성, 복붙 템플릿         | Few-shot 말투 학습 → Browser Use가 DM/이메일 자동 오픈              |

**데모는 소개팅으로 시작하지만, 본질은 "아무 사람이나 웹에서 완벽하게 파악하고, 니치하게 접근할 수 있는" 소셜 인텔리전스 엔진이다.** 연애 매칭($11B)은 킬러 데모이고, 진짜 시장은 인플루언서 마케팅($21B), 세일즈 인텔리전스($200B+)다.

> **심사위원 임팩트 포인트**: "Instagram 하나 긁는 앱"이 아닌, 웹 에이전트가 플랫폼을 넘나들며 사람의 디지털 트윈을 자율적으로 구축하는 기술. Browser Use의 "멀티 사이트 자율 탐색" 능력을 가장 극한까지 보여준다.

## S-2. Problem Statement

### 핵심 문제: 데이팅 앱의 "정보 비대칭"과 "스와이프 피로"

#### 시장 데이터로 본 현실

#### 문제를 3가지 레이어로 분해

**1. 정보의 빈곤 (Root Cause)**

틴더 프로필은 평균 사진 4장 + 자기소개 3줄이다. 이 안에서 상대의 실제 라이프스타일, 유머 코드, 관심사 깊이를 파악하는 것은 불가능하다. 반면 Instagram에는 그 사람의 **실제 생활**이 담겨 있다:

| 데이팅 앱 프로필  | Instagram 공개 활동                                       |
| ----------------- | --------------------------------------------------------- |
| "여행 좋아해요"   | 최근 3개월간 제주도 2회, 교토 1회 방문 (사진으로 확인)    |
| "커피 좋아합니다" | 성수동 스페셜티 카페 12곳 방문 기록, 라떼아트 사진 공유   |
| 정보 없음         | 캡션 말투에서 유머 코드, 존댓말/반말 습관, MBTI 추정 가능 |

**2. 스와이프 피로 (Surface Symptom)**

- Bumble 유료 사용자 8.7% 감소 (4.1M → 3.8M, 2025 Q2)
- 사용자의 48%가 여러 앱을 동시 사용하며 이탈 반복
- "쉽게 만나지만, 특별한 사람을 만나기는 어렵다" — Pew Research

**3. 첫 메시지의 벽 (Conversion Bottleneck)**

매칭 후 실제 대화로 이어지는 비율은 극히 낮다. 대부분 "안녕하세요", "반갑습니다" 수준의 메시지를 보내고, 상대는 수십 개의 비슷한 메시지 중 대부분을 무시한다. **메시지 개인화가 매칭 성공률의 핵심 변수**인데, 이를 자동화할 수단이 없다.

#### 왜 **지금**인가?

```
2024.Q3 ─── Browser Use 오픈소스 공개 → 웹 에이전트 자동화 가능해짐
2024.Q4 ─── Claude Vision 멀티모달 → 이미지에서 라이프스타일 추론 가능
2025    ─── Instagram 3B MAU 돌파 → 지구상 가장 큰 라이프스타일 DB
2025    ─── 데이팅 앱 피로도 정점 (79% 번아웃, 69% 1달 내 삭제)
2026.Q1 ─── SoulMatch → 이 모든 기술을 하나의 에이전트 파이프라인으로 통합
```

Instagram은 30억 사용자의 **라이프스타일 데이터베이스**다. 그러나 이 데이터는 구조화되지 않은 사진, 캡션, 해시태그 형태로 흩어져 있다. Browser Use + Claude Vision의 조합이 이 데이터를 **매칭 가능한 구조화된 프로필**로 변환하는 것을 처음으로 가능하게 했다.

> **심사위원 임팩트 포인트**: "79%가 번아웃인 $11B 시장에서, Instagram 30억 사용자의 비구조화 데이터를 Browser Use가 매칭 엔진으로 변환한다" — 이 한 문장이 40% 비중의 '임팩트' 기준을 정조준한다.

## S-3. Solution Overview

### 핵심 개념: Digital Twin — 웹 상의 나를 완벽하게 이해하는 AI

사람의 온라인 정체성은 하나의 플랫폼에 있지 않다. Instagram에는 라이프스타일, LinkedIn에는 커리어, Facebook에는 인맥과 성향, Twitter/X에는 생각과 가치관이 있다. SoulMatch는 **Browser Use 에이전트가 플랫폼을 자율적으로 넘나들며** 이 조각들을 모아 한 사람의 완전한 디지털 트윈을 생성한다.

```
┌─ BEFORE: 기존 방식 ──────────────────────────────────────────┐
│                                                              │
│  사람을 이해하려면?                                            │
│  → Instagram 들어가서 사진 봄 (라이프스타일 일부만)             │
│  → LinkedIn 검색 (커리어만)                                   │
│  → Google 검색 (단편적)                                      │
│  → 각 플랫폼 따로따로, 수동으로, 30분+                        │
│  → 그래도 빈 곳이 많음 (종합 판단 불가)                       │
│                                                              │
│  😤 "이 사람이 실제로 어떤 사람인지 알 수가 없어"              │
└──────────────────────────────────────────────────────────────┘

                      ▼ SoulMatch ▼

┌─ AFTER: 자율 웹 에이전트가 만드는 디지털 트윈 ───────────────┐
│                                                              │
│  1. Instagram 계정 입력 (30초)                                │
│  2. 🤖 Agent가 IG 탐색 → bio에서 이메일/링크 자동 추출        │
│  3. 🔗 추출된 링크 따라 LinkedIn, Facebook, Twitter 자동 탐색  │
│  4. 🧠 Claude Vision이 모든 플랫폼 데이터 통합 분석           │
│  5. 👤 완벽한 디지털 트윈 생성 (성향/관심사/커리어/가치관)     │
│  6. 💜 디지털 트윈 vs 디지털 트윈 매칭 → 니치한 접근 메시지    │
│  7. 📱 Browser Use가 최적 채널(IG DM/LinkedIn/Email) 자동 오픈│
│                                                              │
│  😍 "이 사람을 웹에서 본 것만으로 이렇게까지 파악할 수 있어?"  │
└──────────────────────────────────────────────────────────────┘
```

### SoulMatch가 해결하는 3가지 핵심 문제

| 문제              | 기존 방식                             | SoulMatch 솔루션                                  | 기술                           |
| ----------------- | ------------------------------------- | ------------------------------------------------- | ------------------------------ |
| **정보의 조각화** | 각 SNS 따로 검색, 수동 종합           | 크로스 플랫폼 자동 탐색 → 디지털 트윈 통합        | Browser Use 멀티 사이트 크롤링 |
| **피상적 이해**   | 프로필 사진과 자기소개만              | 사진+캡션+커리어+가치관+관심사 입체 분석          | Claude Vision + 멀티모달 분석  |
| **접근의 벽**     | "안녕하세요" 콜드 메시지 (응답률 <5%) | 디지털 트윈 기반 초개인화 메시지 + 최적 채널 선택 | Few-shot + 자율 채널 판단      |

### 차별화 포인트: "디지털 트윈 = 소셜 인텔리전스의 원자 단위"

SoulMatch의 핵심 혁신은 **"디지털 트윈"이라는 새로운 데이터 자산**을 만드는 것이다.

1. **Cross-Platform Autonomous Crawling**: IG에서 링크를 발견하면 자율적으로 LinkedIn, Twitter까지 탐색 — 사람이 하면 30분, 에이전트는 1분
2. **Digital Twin as Data Asset**: 흩어진 소셜 데이터를 하나의 구조화된 "디지털 트윈"으로 변환 — 이것 자체가 가치
3. **Multi-Channel Smart Outreach**: 분석만 하는 게 아니라, "이 사람에게는 LinkedIn으로 접근하세요"까지 판단하고 실행
4. **Hook = Dating, Moat = Intelligence**: 소개팅 앱으로 시작하지만, 디지털 트윈 기술은 마케팅/세일즈/채용 어디든 적용

```
기존 도구:      하나의 플랫폼 → 부분 분석 → 끝
SoulMatch:      크로스 플랫폼 자율 탐색 → 디지털 트윈 생성 → 매칭 → 최적 채널 접근 → 행동까지
```

### SoulMatch의 5가지 구조적 경쟁 우위

**1. 데이터 모트 (Data Moat)**

- 기존 앱: 사용자가 직접 입력한 "꾸며진" 데이터
- SoulMatch: Instagram의 실제 활동 데이터 → **진정성의 격차**
- 사용자가 프로필을 꾸밀 수 없음 → 매칭 품질의 근본적 차이

**2. 제로 프로필 작성 (Zero Profile Effort)**

- 기존 앱: 프로필 작성 15분 + 사진 선택 + 자기소개 고민
- SoulMatch: Instagram 계정 입력 30초 → 끝
- **온보딩 마찰을 90% 이상 제거** → 전환율 극대화

**3. 플랫폼 불가지론 (Platform Agnostic)**

- 기존 앱: 자체 폐쇄 생태계 안에서만 매칭
- SoulMatch: Browser Use로 어떤 소셜 플랫폼이든 접근 가능
- Instagram → Twitter/X → LinkedIn → TikTok 순차 확장

**4. End-to-End 자동화**

- 기존 앱: 매칭까지만, 대화는 사용자 몫
- SoulMatch: 수집 → 분석 → 매칭 → 메시지 생성 → DM 오픈
- **"행동까지 하는 에이전트"** — 분석 도구가 아닌 실행 도구

**5. B2B 피봇 가능성**

- 기존 앱: B2C 데이팅에 갇혀 있음
- SoulMatch: 동일 기술 → 인플루언서 발굴, HR 리크루팅, 세일즈 리드 발굴
- **$11B 데이팅 → $21B 인플루언서 마케팅 → $200B+ HR/세일즈**

## P-2. Feature Specification

### Feature 1: Instagram 자동 프로필 수집

| 항목            | 상세                                                                                                                                            |
| --------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| **기능명**      | Profile Collector                                                                                                                               |
| **트리거**      | 사용자가 @username 입력 + "분석 시작" 클릭                                                                                                      |
| **입력**        | Instagram username (string)                                                                                                                     |
| **처리**        | Browser Use Agent가 `instagram.com/{username}` 자동 탐색                                                                                        |
| **출력**        | `ProfileData` JSON (bio, image_urls[], captions[], hashtags[], post_frequency, is_private)                                                      |
| **성공 기준**   | 공개 계정: 최근 게시물 최대 20개 데이터 수집 완료                                                                                               |
| **실패 처리**   | 비공개 계정: `is_private=true` 반환, 분석 불가 안내. 네트워크 오류: 3회 지수 백오프 재시도 (2초→4초→8초). 최종 실패: 빈 프로필 반환 + 에러 안내 |
| **엣지 케이스** | 게시물 0개 계정 → bio만 분석. 비공개 계정 → 즉시 안내 + 데모 모드 제안. Instagram 로그인 필요 → 세션 프로필로 자동 처리                         |
| **구현 상태**   | ✅ 완전 구현 (`backend/agents/profile_collector.py`)                                                                                            |

### Feature 2: 멀티모달 AI 프로필 분석

| 항목          | 상세                                                                                                                                                          |
| ------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **기능명**    | Multimodal Analyzer                                                                                                                                           |
| **트리거**    | Profile Collector 완료 후 자동 실행                                                                                                                           |
| **입력**      | ProfileData (이미지 URL[], 캡션[], 해시태그[], bio)                                                                                                           |
| **처리**      | 3방향 병렬 분석: ① Claude Vision 이미지 분석 (최대 12장) ② Claude 텍스트/캡션 분석 ③ Tavily 해시태그 서치 보강                                                |
| **출력**      | 시각 분석 dict + 텍스트 분석 dict + 서치 보강 dict                                                                                                            |
| **성공 기준** | 5개 이상의 니치 관심사 태그 추출 + 커뮤니케이션 스타일 분류                                                                                                   |
| **실패 처리** | API 키 없음: Mock 분석 결과 자동 반환 (username 시드 기반 일관된 관심사). Claude 타임아웃: 텍스트 분석만으로 폴백. Tavily 실패: 이미지+텍스트 분석만으로 진행 |
| **한국 특화** | 성수동/한남동/연남동 위치 인식, K-패션 스타일 분류, MZ세대 언어 패턴 (ㅋㅋ, ~임, 이모지 패턴)                                                                 |
| **구현 상태** | ✅ 완전 구현 (`backend/agents/multimodal_analyzer.py`)                                                                                                        |

### Feature 3: AI 매칭 엔진

| 항목            | 상세                                                                                       |
| --------------- | ------------------------------------------------------------------------------------------ |
| **기능명**      | Matching Engine                                                                            |
| **트리거**      | Profile Builder 완료 후 / GET /api/matches 호출                                            |
| **입력**        | 사용자 프로필 임베딩 (1536dim) + 후보자 프로필 임베딩                                      |
| **처리**        | 코사인 유사도 (60%) + 관심사 키워드 겹침 (40%) 가중 합산                                   |
| **출력**        | `MatchCard[]` — 상위 5~10명, 점수 0.75~0.98 범위 정규화                                    |
| **성공 기준**   | 매칭 점수 상위 추천이 실제 공통 관심사와 일관성 있음                                       |
| **실패 처리**   | DB 없음/프로필 부족: SAMPLE_MATCHES 6명 하드코딩 반환                                      |
| **관심사 매칭** | 부분 문자열 매칭 (예: "커피" ∩ "스페셜티커피" = hit)                                       |
| **구현 상태**   | ✅ 엔진 구현 완료 (`backend/core/matching_engine.py`). 현재 MVP는 SAMPLE_MATCHES 직접 반환 |

### Feature 4: AI 개인화 메시지 생성

| 항목             | 상세                                                                                                                                  |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| **기능명**       | Message Generator                                                                                                                     |
| **트리거**       | 사용자가 매칭 카드 우스와이프 (LIKE)                                                                                                  |
| **입력**         | `MessageRequest` — sender_username, target_username, common_interests[]                                                               |
| **처리**         | 2단계 파이프라인: STEP 1) 사용자 캡션 말투 분석 (존댓말/반말, 톤, 이모지) → STEP 2) 스타일 + 공통 관심사 기반 메시지 3개 생성         |
| **출력**         | `MessageResponse` — messages[3], 각 50자 이내                                                                                         |
| **3가지 접근법** | ① 공통 장소 언급형 ② 관심사 질문형 ③ 감성 공감형                                                                                      |
| **성공 기준**    | 메시지가 사용자 말투와 일치 + 공통 관심사 자연스럽게 포함                                                                             |
| **실패 처리**    | API 키 없음: 8카테고리(등산/카페/전시/독서/운동/요리/사진/필라테스) × 3개 = 24개 한국어 fallback 템플릿에서 키워드 부분 매칭으로 선택 |
| **구현 상태**    | ✅ 완전 구현 (`backend/agents/message_generator.py`)                                                                                  |

| 항목          | 상세                                                                                                     |
| ------------- | -------------------------------------------------------------------------------------------------------- |
| **기능명**    | DM Opener                                                                                                |
| **트리거**    | 사용자가 메시지 선택 + "이 메시지로 DM 보내기" 클릭                                                      |
| **입력**      | `OpenDMRequest` — target_username, message                                                               |
| **처리**      | Browser Use Agent가 Instagram DM 화면 탐색 → username 검색 → DM창 오픈 → 메시지 텍스트 입력              |
| **출력**      | `OpenDMResponse` — success: bool, message: string                                                        |
| **안전 장치** | ⚠️ **전송 버튼 절대 클릭하지 않음** — 에이전트 프롬프트에 명시. 사용자가 Instagram에서 직접 확인 후 전송 |
| **성공 기준** | DM 대화창 오픈 + 메시지 입력 완료 (`DM_READY` 문자열 감지)                                               |
| **실패 처리** | Instagram 로그인 필요: 세션 프로필 자동 로드. DM 제한: 에러 메시지 반환. Mock 모드: 즉시 성공 응답       |
| **구현 상태** | ✅ 완전 구현 (`backend/agents/dm_opener.py`)                                                             |

> **심사위원 임팩트 포인트**: 5개 기능 모두 "실패해도 graceful"한 설계. API 키 없는 데모 환경에서도 Mock 모드로 전체 플로우 시연 가능 — 라이브 데모 실패 리스크 최소화.

```
    error: string | null       → 에러 메시지

/match:
  matches: MatchCard[]         → API에서 가져온 매칭 카드
  currentIndex: number         → 현재 보고 있는 카드 인덱스
```

```python
    captions: list[str]        # 각 게시물 캡션
    hashtags: list[str]        # 사용된 해시태그 전체
    post_frequency: int        # 주간 게시 빈도
    is_private: bool           # 비공개 계정 여부

class AnalysisResult(BaseModel):
    """AI 분석 결과"""
    interests: list[str]       # ["스페셜티커피매니아", "주말등산러", ...]
    lifestyle: dict            # {energy: "high", social: "active", ...}
    personality_big5: dict     # {openness: 0.8, conscientiousness: 0.6, ...}
    communication_style: str   # "반말 위트체, 이모지 활발"

class MatchCard(BaseModel):
    """매칭 추천 카드"""
    username: str
    profile_image_urls: list[str]
    common_interests: list[str]
    ai_summary: str            # AI가 생성한 2~3줄 요약
    compatibility_score: float # 0.75 ~ 0.98
    is_private: bool = False

class AnalyzeProfileRequest(BaseModel):
    username: str              # Instagram @username

class AnalysisStatusResponse(BaseModel):
    username: str
    status: str                # "processing" | "completed" | "failed"
    progress: int              # 0 ~ 100
    current_step: str          # "프로필 수집 중..." 등
    result: Optional[AnalysisResult] = None

class MessageRequest(BaseModel):
    sender_username: str
    target_username: str
    common_interests: list[str]

class MessageResponse(BaseModel):
    messages: list[str]        # 3개 메시지
    target_username: str

class OpenDMRequest(BaseModel):
    target_username: str
    message: str               # 선택된 메시지 텍스트

class OpenDMResponse(BaseModel):
    success: bool
    message: str               # 성공/실패 메시지
```

## B-5. Q&A Cheat Sheet

### 심사위원 예상 질문 20개 + 킬러 답변

**[임팩트 — 40%]**

**Q1: "이거 실제로 사용할 사람이 있나요?"**

> 79%의 데이팅 앱 사용자가 번아웃을 경험하고 있습니다. 그들이 원하는 건 "더 나은 스와이프"가 아니라 "진짜 맞는 사람을 아는 것"이에요. SoulMatch는 Instagram이라는 이미 존재하는 라이프스타일 데이터를 활용해 그 니즈를 해결합니다.

**Q2: "시장이 얼마나 큰가요?"**

> 온라인 데이팅: $11B (2025). 인플루언서 마케팅: $21B. HR/세일즈 인텔리전스: $200B+. SoulMatch의 5-Agent Pipeline은 "소셜 프로필 분석 + 개인화 아웃리치"의 범용 엔진이라 어디든 적용됩니다.

**Q3: "어떻게 돈을 벌 건가요?"**

> B2C Pro 구독 $9.99/월 (마진 87%). B2B 인플루언서 발굴 $49/월. 장기적으로 API + 화이트라벨 Enterprise.

**Q4: "사용자가 왜 다른 데이팅 앱 대신 이걸 쓰나요?"**

> 프로필 작성 0초 (Instagram 계정이 곧 프로필), AI가 분석한 "이유가 있는 매칭", 내 말투로 학습한 첫 메시지까지. 기존 앱은 "스와이프 반복"이고, SoulMatch는 "AI가 준비해주는 소개팅"입니다.

---

**[창의성 — 20%]**

**Q5: "Browser Use를 왜 이 유스케이스에 선택했나요?"**

> SoulMatch의 핵심은 **크로스 플랫폼 자율 탐색**입니다. Instagram에서 시작해 bio의 링크를 발견하면 자율적으로 LinkedIn, Twitter로 이동합니다. 이건 API로는 절대 불가능해요 — 에이전트가 "사람처럼" 브라우저를 넘나들어야 합니다. Browser Use만이 이걸 할 수 있어요. 게다가 수집뿐 아니라 DM 오픈까지 — 분석에서 행동까지 하나의 에이전트 체인으로 해결합니다.

**Q6: "이건 그냥 웹 스크래핑 아닌가요?"**

> 웹 스크래핑은 한 사이트에서 데이터를 가져오기만 합니다. SoulMatch 에이전트는 Instagram에서 링크를 발견하면 **자율적으로 LinkedIn으로 이동하고, 거기서 또 Twitter를 찾아갑니다.** 그리고 수집한 멀티 플랫폼 데이터를 Claude Vision으로 통합 분석해 "디지털 트윈"을 만들고, 최적 채널로 개인화 접근까지 실행합니다. 스크래핑이 아니라 자율 판단하는 소셜 인텔리전스 에이전트예요.

**Q7: "Clearbit이나 Apollo랑 뭐가 다른가요?"**

> Clearbit/Apollo는 **회사 데이터**를 구조화합니다. SoulMatch는 **사람 데이터**를 구조화해요. 회사의 매출, 직원 수가 아니라 한 사람의 라이프스타일, 가치관, 커뮤니케이션 스타일을 크로스 플랫폼으로 파악합니다. 그리고 이들은 분석만 하지만, SoulMatch는 분석 + 개인화 메시지 생성 + 최적 채널 접근까지 end-to-end로 실행합니다.

---

**[기술 난이도 — 20%]**

**Q8: "기술적으로 가장 어려웠던 점은?"**

> 5개 에이전트 간 오케스트레이션입니다. Browser Use 수집 → Claude Vision 분석 → 임베딩 생성 → 말투 학습 → 다시 Browser Use DM 오픈까지, 각 에이전트가 다른 AI 서비스를 사용하면서 데이터를 체인으로 전달해야 합니다. 어느 한 단계가 실패해도 전체가 graceful하게 동작하도록 설계했어요.

**Q9: "AI 분석 정확도는?"**

> Claude Vision이 사진에서 장소, 음식, 활동을 추론하고, 캡션에서 성격을 분석합니다. 100% 정확하진 않지만, 데이팅 앱 프로필 3줄보다 훨씬 풍부한 정보를 제공합니다. 그리고 사용자가 "AI 분석 더 보기"로 결과를 투명하게 확인할 수 있어요.

**Q10: "실시간으로 Instagram을 탐색하는 게 느리지 않나요?"**

> Browser Use Cloud의 세션 프로필로 Instagram 로그인 상태를 캐싱합니다. 실제 수집은 15~30초. 그 동안 파티클 배경 + 5단계 진행 상황 애니메이션으로 사용자 경험을 유지합니다. 그리고 "느린 것 자체가 WOW" — 에이전트가 실시간으로 브라우저를 조작하는 모습이 데모 포인트입니다.

**Q11: "인터넷 안 되면 어떡하나요?"**

> Plan A/B/C 3중 백업이 있습니다. Mock 모드가 이미 구현되어 있어서, API 키 없이 로컬 서버만으로 전체 플로우를 시연할 수 있어요. 백업 비디오도 준비합니다.

**Q12: "데모에서 보여준 건 실제로 동작하는 건가요?"**

**Q14: "Instagram이 차단하면?"**

**Q15: "GDPR 준수는?"**

> GDPR Art. 6(1)(a) 명시적 동의, Art. 17 삭제권, Art. 5(1)(c) 데이터 최소화 원칙 모두 설계에 반영했습니다. 온보딩에서 3단계 동의를 받고, opt-out 페이지를 운영합니다.
